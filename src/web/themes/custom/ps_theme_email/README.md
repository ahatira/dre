# ps_theme_email

MJML theme for Property Search transactional emails (`engine: twig_mjml`).

## Preview (dev)

Requires `mjml_render_devel` (enabled via config split `local` or `ps_core` install).

**Access:** permission `preview mjml templates` — granted to `administrator` and `site_admin` (`make rbac-sync`).

Preview UI: `/admin/config/development/mjml-preview` (must be logged in).

**Variables:** `ps_email` injects branding, design tokens and rich footer into every
`ps_theme_email` preview via `EmailMjmlPreviewVariablesBuilder` (same data as live
sends). Module-specific sample data can extend via `hook_ps_email_mjml_preview_variables_alter()`.
Manual overrides in the MJML Devel form still take precedence.

**Note:** production sends use `ps_theme/templates/email-wrap.html.twig` (HTML). MJML
preview shows `ps_theme_email` templates — structure and branding match; minor markup
differences are expected between MJML and hand-written HTML shells.

**Email hub (Phase 6):** `/admin/ps/config/email` (`ps_email`) — registry of all transactional types, Mailpit tests and module settings links.

| Template | Purpose |
|----------|---------|
| `email-shell-preview` | Full shell (`email-wrap`) |
| `email-partials-preview` | Partials showcase (button, card, alert, table) |
| `email-contact-preview` | Contact hub confirmation sample |
| `email-compare-preview` | Compare share sample (CTA + table) |
| `email-search-alert-preview` | Search alert digest sample |
| `email-import-alert-preview` | CRM import pipeline failure alert sample |

## Stack (no Easy Email)

Transactional emails use **Symfony Mailer** + **MJML** (`ps_theme_email`). Easy Email is **not** part of this project — core user mails keep Drupal defaults unless overridden later in code.

## Production sends

Symfony Mailer uses `ps_theme_email` for **webform** (contact), **ps_compare**, **ps_search** (alert digest), and **ps_migrate** (import pipeline alerts).

## Partials

Located in `templates/partials/` (extension `.html.twig` — not listed individually in MJML Devel, use `email-partials-preview`).

| Partial | Variables |
|---------|-----------|
| `header` | logo, slogan (from preprocess) |
| `title` | subject, contact confirmation title |
| `button` | `button_url`, `button_label` |
| `card` | `card_title`, `card_body`, optional footer link |
| `alert` | `alert_message`, `alert_variant` (success/info/warning) |
| `table` | `table_columns`, `table_sections` (compare model) |
| `signoff` | team name, site URL |
| `footer-default` | legal footer line |

Include from MJML templates:

```twig
{% include '@ps_theme_email/partials/button.html.twig' with { button_url: url, button_label: label } only %}
```
